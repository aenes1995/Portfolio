About Me

I have always had a strong desire to create. Fortunately, I studied computer engineering. This decision guided me to new horizons such as programming and computer science.

In addition to these, I have loved playing computer games since childhood. I have always been fascinated by the technology and imagination behind them. This fascination made me dream of creating my own games and showing them to my friends. All of these experiences pushed me toward game development.

Game Development

Tapp!

Google Play · Link

It's a published 2D mobile arcade game. I handled the gameplay, UI design, music, as well as game and level design. It also includes many subsystems for maintenance and an improved user experience.

WS

It is a 3D mobile zombie survival game. I created the gameplay, SFX, and both level and game design. We needed to develop a level design tool to allow this project to progress properly.

This tool can generate levels from our data—such as which actions occur in a level or which enemies appear. It can also decorate rooms and place props in appropriate areas.

Due to financial constraints, we could not continue development of this game.

Features:

  • Procedural Generation
  • Shader Graph
  • Post Processing

Procedural Tree & Vegetation Generator

I designed a tree-generation method for my dream game. I used only cubes to create the plants because of an artistic choice I made. I can say that my method works quite well in some scenarios, but there is still room for improvement.

I also designed an editor to generate vegetation. It uses ECS to handle large numbers of instances efficiently.

As a note I used this Git repo to generate low poly terrains.

Features:

  • Procedural Generation
  • ECS (Entity Component System)
  • Shader Graph
  • Post Processing

3D Locomotion & Combat AI

Empty Paper - GGJ 2015

Event Page · Link

It was my freshman year in college. A couple of my friends and I decided to participate in GGJ 2015. It was also my first game jam. Each of us took on a role for the event.
